## 关于Carson

> Carson永远沉溺于过去和未来。

或许Carson很早就习惯了这种悬置感。

许多时候，他并不稳稳地站在某一个确定的当下，而是更容易被已经发生过的东西牵引，也更容易被那些尚未到来的情景召唤。

过去在他身上留下痕迹，未来向他投来影子，而现在常常只是两者短暂交汇的一小块地面。他站在上面，停留，观察，记录，然后继续往前。

他学软件工程，也因此拥有了一种偏理性的观察方式。

系统、结构、逻辑、MVP、实现路径，这些东西塑造了他理解世界的一部分方法。可他并没有在这些词语里停下来太久。比起一个系统能够做什么，他越来越在意它如何被人触碰，如何被人误解，如何在一次点击、一次停顿、一次迟疑里显露出自身的性格。技术在纸面上总是整齐的，落到真实生活里却会变形，会沉默，会暴露出许多更细微的问题。那些问题后来比答案更吸引他。

他曾经也被更大的叙述打动过。那些关于方向、时代、效率、变革的词语，一度像远处明亮的灯，让人愿意沿着它们指向的地方奔跑。只是慢慢地，他对宏大的句子变得谨慎起来。相比那些高悬在空中的表达，他开始更相信局部，相信细节，相信那些不那么响亮却更贴近人的时刻。一个界面是否清晰，一段反馈是否温和，一种交互是否真的照顾到了使用者的犹豫与专注，这些问题在他心里的分量一点一点变重。比起谈论技术将把人带向哪里，他更关心技术此刻如何与人相处。

有一段时间，他把自己放进真实的教学现场里。课堂、讲台、屏幕、教师的话语、学生的目光、那些顺利与不顺利的瞬间，共同构成了一种与想象完全不同的密度。很多抽象的判断在那里被重新校准。技术并不会因为名字新、概念大就自动成立，它需要被人理解，被人信任，被人愿意纳入自己的工作节奏；它也需要承受误差、摩擦和现实的迟缓。那段经历后来渐渐成为一段已经合上的章节，留在他身上的，却不是某个简单的结论，而是一种更安静的确信：真正重要的东西，往往发生在尺度更小、更具体、也更难被口号覆盖的地方。

于是他的目光开始向内收，也向近处移动。与其不断追问更大的系统，更广的叙事，更遥远的未来，他更愿意把注意力放在界面、注意力、感知、认知、学习过程、人与工具之间那些微妙的接缝上。他开始对“如何设计一种不打断人的技术”这样的问题生出兴趣，也开始反复想着，什么样的产品能够既有效，又不过分冒犯人的感受；既足够聪明，又保留某种朴素的分寸感。很多时候，他所寻找的已经不再是声势，而是准确。

他的性格里始终有两股并行的力量。一股把他推向秩序，推向结构，推向那些可以被拆解、被理解、被重新搭建的东西；另一股则把他推向更难命名的部分，推向情绪、推向记忆、推向那些尚未说清但真实存在的感受。他听很多摇滚乐，也乐于制造一点属于自己的声音；他会为某些球队的胜负牵动情绪，也会因为一本书里的某一句话停下来很久。看上去互不相关的事物，最后都在他身上汇成同一种倾向：他对“人是如何感受世界的”始终怀有耐心。

Carson并不急于把自己定义得太完整。完整有时意味着被定义，而他还愿意保留一些未完成。比起稳定地成为某一种人，他似乎更习惯于在很多个方向之间缓慢移动：在技术和表达之间，在研究和产品之间，在理性与敏感之间，在已经经历过的生活与尚未抵达的地方之间。他正在慢慢接纳这种移动，也逐渐学会从中辨认自己。

他喜欢那些真正有用的东西。不是喧闹的功能堆叠，也不是用力过猛的聪明感，而是那种经过思考之后仍然显得克制、平静、顺手的设计。一个好的工具应该让人感觉自己被理解，而不是被教育；应该帮助人把注意力放回问题本身，而不是放在工具自己的存在感上。这种偏好延伸到了他看待许多事物的方式里：他偏爱有边界感的表达，偏爱不夸张的判断，偏爱那些说到恰好为止的句子。

至于未来，他依然会想得很多。那种对远方的感受并没有消失，只是变得比从前更安静了一些。它不再总是以宏大的想象出现，更多时候，它藏在一些很小的愿望里：希望自己能继续靠近真正重要的问题，继续做出更有分寸感的东西，继续在技术与人之间寻找一种更长久的平衡，继续在每一次构建与每一次书写里，把模糊的感受慢慢整理清楚。很快，他也将去往斯德哥尔摩，在 KTH 的 Interactive Media Technology 项目里继续自己的下一段学习。那并不只是一次地理意义上的迁移，更像是他此前那些兴趣与犹疑、结构与感受、技术与人的问题，终于在另一座城市、另一种媒介语境里，找到了继续生长下去的空间。

而过去也不会真的离开。他仍然会反复回望一些阶段、一些城市、一些相遇、一些曾让他确信自己正在靠近某种真实的瞬间。只是如今的回望里少了一点执拗，多了一点理解。那些已经结束的部分并没有消失，它们只是沉到更深处，变成了观看方式，变成了判断，变成了他后来面对新世界时的底色。

“Carson永远沉溺于过去和未来。”

只是现在，他也开始愿意把更多目光留给那些具体的人、具体的界面、具体的时刻。宏大的叙述渐渐退到远处，真正留在手边的，是一次次更小、更近、也更诚实的接触。对他来说，很多事情正是从这里开始变得重要。

## 关于Coding

对 Carson 来说，Coding 是一种组织复杂性的方式。

他以 Python 为主要语言，长期用于数据分析、机器学习与实验验证；也使用 JavaScript / TypeScript 与 React 进行前端开发，关注页面结构、组件逻辑与交互节奏。

在机器学习与深度学习方向，他接触过从传统模型到神经网络模型的一系列方法，包括 SVM、XGBoost、CNN、RNN、GNN 与 Transformer，也使用过 PCA、LASSO、SHAP、Grad-CAM、LIME 等方法处理特征、解释结果与理解模型。

在生成式 AI 方向，他关注 Prompt Engineering、RAG、Agent 与 AI 应用编排，也尝试过 Dify、Coze、LangChain、AutoGen Studio 等工具。相比单纯调用模型，他更关心 AI 如何进入真实任务流程，如何被组织成稳定、可解释、可持续使用的产品能力。

他也始终保留着对人机交互与产品设计的兴趣。Figma、Axure、XMind，以及 SWOT、KANO 模型、用户体验五要素等方法，构成了他理解产品的另一条路径。

代码在他这里通向更具体的东西：界面、体验、产品，以及人与技术之间那些微妙的接缝。

## 关于本站

这个网站是 Carson 为自己搭建的一处缓慢更新的空间。

它既不是作品集的陈列柜，也不只是简历的网页版本。它更像一个可以被持续修改的个人界面：用来放置文字、项目、实验、照片、声音，以及一些尚未被整理成明确结论的想法。

本站目前基于 React / TypeScript 构建，使用组件化的方式组织页面结构，并结合 Framer Motion 处理细微的动效与转场。样式上尽量保持克制：较窄的正文栏、明确的留白、低噪声的视觉层级，以及少量带有叙事意味的交互细节。

Carson 希望它看起来不像一个急于展示自己的地方，而像一间仍在整理中的房间。这里会有技术，也会有文字；会有完成的项目，也会有未完成的观察。它记录一个人如何使用工具、理解世界，也记录一个人如何在过去与未来之间，慢慢把自己安放到当下。
